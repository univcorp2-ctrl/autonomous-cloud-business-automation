# 安全性・規約・コンプライアンス

このテンプレートは自動化の土台ですが、実運用では安全性と規約確認が必須です。

## データ収集

- スクレイピングより、正規 API / データ提供契約 / 許諾済みデータを優先する。
- robots.txt、利用規約、アクセス頻度制限を確認する。
- 個人情報や非公開情報を保存しない。

## 物販・転売

- マーケットプレイスの規約を確認する。
- 自動購入・自動出品・価格操作は規約違反や法的問題になる可能性がある。
- 初期状態では候補抽出とシミュレーションのみ行う。

## 株式・暗号資産

- 初期状態では paper trading / backtest のみ。
- ライブ取引を解禁する場合は、証券会社・取引所の正式 API と規約を確認する。
- 損失上限、発注上限、緊急停止、監査ログ、人間承認を必須にする。
- AI判定だけで発注しない。

## Secrets 管理

- APIキーは GitHub Secrets または Cloudflare Secrets に保存する。
- コードや README に直接書かない。
- 権限は最小限にする。
- 定期的にローテーションする。

## Custom GPT Action の保護

- Bearer Token を使う。
- 重要操作は `dryRun=true` を初期値にする。
- ライブ取引、本番発注、削除系操作は追加承認を要求する。
- すべての実行依頼をログに残す。
