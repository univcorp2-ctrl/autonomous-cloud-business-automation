# ユースケース別の実装方針

## 1. 不動産物件の情報収集・分析

### 入力

- エリア
- 価格帯
- 駅距離
- 築年数
- 利回り条件
- 取得元 API / 許諾済みデータソース

### 処理

1. 物件情報を取得。
2. 重複を除外。
3. 価格、賃料、利回り、周辺相場を計算。
4. スコアリング。
5. Web 画面と CSV に出力。

### 出力例

- 想定利回り
- 周辺相場との差分
- 修繕リスク
- 駅距離ペナルティ
- おすすめ度

## 2. 物販・転売候補の整理

### 入力

- 商品カテゴリ
- 仕入れ上限価格
- 販売先
- 手数料
- 送料
- 在庫条件

### 処理

1. 正規 API や許諾済みデータから価格を取得。
2. 仕入れ価格、販売価格、手数料、送料から利益を計算。
3. 在庫、回転率、競合数を加味してスコアリング。
4. CSV / Web ダッシュボードに表示。

### 注意

- 各マーケットプレイスの利用規約を確認する。
- 自動購入・自動出品は規約違反になる可能性があるため、初期状態では実装しない。

## 3. 株式投資・暗号資産の自動売買シミュレーション

### 入力

- 銘柄 / 通貨ペア
- 期間
- 戦略
- 資金量
- 手数料
- 損切りルール

### 処理

1. 価格データを取得。
2. 売買シグナルを生成。
3. バックテストを実行。
4. 損益、最大ドローダウン、勝率、シャープレシオ相当の指標を表示。
5. Web / CSV に出力。

### ライブ取引に進む条件

- 証券会社・取引所の正式 API を使用する。
- paper trading で十分に検証する。
- 発注上限、損失上限、停止条件を設定する。
- 監査ログを残す。
- 人間の承認フローを残す。

## 4. Excel / CSV 出力

このテンプレートでは `/api/export.csv` を用意しています。Excel では CSV を開けば確認できます。

将来的に `.xlsx` を直接生成したい場合は、以下のどちらかにします。

- Node.js バックエンドで ExcelJS 等を使う。
- Cloudflare Worker では CSV を生成し、必要に応じて外部ジョブで xlsx 化する。

## 5. AIによる最終チェック

AIレビューは補助用途です。

- E2E のスクリーンショットを確認する。
- 表示内容が空でないか確認する。
- 異常値や桁違いの値を検出する。
- ログにエラーがないか要約する。

ただし、AI判定だけで本番売買や本番発注を許可しないでください。
